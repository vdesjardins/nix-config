# Neovim Cmdline Navigation - Arrow Keys Fix

## Prompt
Fix arrow key navigation in Neovim cmdline. Previous implementation of keymappings.nix wasn't being applied. Need to use Lua directly for arrow key and Tab navigation in cmdline completion menu.

## Changes Made

### 1. Removed Cmdline Keymappings from keymappings.nix
**File**: `home/modules/modules/desktop/editors/nixvim/keymappings.nix`

Removed the nixvim-style cmdline mode keybindings that weren't being generated correctly. Arrow keys remain disabled in normal and insert modes as intended.

### 2. Implemented Arrow Keys via Lua in blink-cmp.nix
**File**: `home/modules/modules/desktop/editors/nixvim/plugins/blink-cmp.nix`

Enhanced `extraConfigLua` with proper Lua keymappings using `vim.keymap.set()`:
- **`<Up>`**: Calls `cmp.select_prev()` if menu visible, otherwise feeds Up arrow normally
- **`<Down>`**: Calls `cmp.select_next()` if menu visible, otherwise feeds Down arrow normally
- **`<Tab>`**: Calls `cmp.select_next()` if menu visible, otherwise feeds Tab normally
- **`<S-Tab>`**: Calls `cmp.select_prev()` if menu visible, otherwise feeds Shift+Tab normally

Uses `vim.feedkeys()` with `nvim_replace_termcodes()` to properly handle fallback behavior when completion menu is not visible.

## Result
- ✅ Arrow keys now navigate completion menu in cmdline
- ✅ Tab/Shift+Tab scroll through completions  
- ✅ Enter selects highlighted completion or executes command
- ✅ All pre-commit checks passed
- ✅ Home-manager configuration applied successfully
- ✅ Lua keymaps properly generated in ~/.config/nvim/init.lua

## Testing
To verify the changes work:
1. Open Neovim
2. Enter command-line mode (`:`)
3. Type a command to trigger completions (e.g., `:set `)
4. Use `<Up>`/`<Down>` arrows to navigate menu options
5. Use `<Tab>`/`<S-Tab>` to scroll through options
6. Press `<CR>` to select the highlighted option

## Technical Details
- Uses `cmp.is_visible()` to check if completion menu is displayed
- Uses `cmp.select_next()` and `cmp.select_prev()` to navigate selections
- Uses `vim.feedkeys()` + `nvim_replace_termcodes()` to handle normal key behavior when menu not visible
- Keymaps set with `noremap = true` to prevent recursive expansion
