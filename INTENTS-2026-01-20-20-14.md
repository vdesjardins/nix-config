# Neovim Cmdline Navigation Configuration

## Prompt
Configure Neovim cmdline to use arrow keys for navigation, tab for scrolling through completions, and enter to select choices.

## Changes Made

### 1. Enable Arrow Keys in Cmdline Mode
**File**: `home/modules/modules/desktop/editors/nixvim/keymappings.nix`

Added cmdline-mode specific mappings:
- `<Up>` → `<C-p>` (navigate to previous completion/history)
- `<Down>` → `<C-n>` (navigate to next completion/history)
- `<Left>` → `<Left>` (move cursor left in command)
- `<Right>` → `<Right>` (move cursor right in command)

Arrow keys remain disabled in normal and insert modes as before.

### 2. Add Tab/Shift+Tab Navigation for Cmdline Completions
**File**: `home/modules/modules/desktop/editors/nixvim/plugins/blink-cmp.nix`

Enhanced `extraConfigLua` with:
- Tab key: Scrolls to next completion if menu is visible, otherwise inserts tab
- Shift+Tab: Scrolls to previous completion if menu is visible, otherwise inserts tab
- Uses `expr` mode for proper key binding evaluation in cmdline context

## Result
- ✅ Arrow keys now navigate completion menu in cmdline
- ✅ Tab scrolls forward through completions
- ✅ Shift+Tab scrolls backward through completions
- ✅ Enter still selects the highlighted completion or executes the command
- ✅ All pre-commit checks passed
- ✅ Home-manager configuration applied successfully

## Testing
To verify the changes work:
1. Open Neovim
2. Enter command-line mode (`:`)
3. Type a command to trigger completions
4. Use `<Up>`/`<Down>` arrows to navigate
5. Use `<Tab>`/`<S-Tab>` to scroll through options
6. Press `<CR>` to select
