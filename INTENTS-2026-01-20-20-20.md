# Neovim Cmdline - Fix Arrow Keys for History Navigation

## Prompt
Arrow keys were working in the completion menu but throwing an error when the cmdline is empty. Need to preserve default cmdline behavior (previous/next command history) when completion menu is not visible.

## Changes Made

### Enhanced Arrow Key Handling in blink-cmp.nix
**File**: `home/modules/modules/desktop/editors/nixvim/plugins/blink-cmp.nix`

Modified `extraConfigLua` to use proper `vim.api.nvim_feedkeys()` with mode `'n'` for normal key processing:

- **`<Up>`**: 
  - If completion menu visible → navigate previous item
  - If menu hidden → feed `<C-p>` (previous command history)
  
- **`<Down>`**: 
  - If completion menu visible → navigate next item
  - If menu hidden → feed `<C-n>` (next command history)

- **`<Tab>` / `<S-Tab>`**: 
  - If completion menu visible → navigate menu
  - If menu hidden → default tab behavior

Uses `vim.api.nvim_feedkeys()` with mode `'n'` and `false` flag to properly process keys without triggering nested keymaps.

## Result
- ✅ Arrow keys navigate completion menu when visible
- ✅ Arrow keys access command history when menu is hidden (no errors)
- ✅ Tab/Shift+Tab work correctly in both scenarios
- ✅ All pre-commit checks passed
- ✅ Home-manager configuration applied successfully
